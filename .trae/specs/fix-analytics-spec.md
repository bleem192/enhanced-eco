# 数据分析和销售统计功能修复 - 产品需求文档

## 概述
- **摘要**：修复管理后台销售业绩排行统计不准确、以及数据分析页面热门商品和活跃用户数据缺失的问题
- **目的**：确保管理后台的销售业绩排行能够正确统计每个销售负责的订单，数据分析页面能够显示完整的销售趋势、商品分类占比、热门商品和活跃用户数据
- **目标用户**：系统管理员

## 问题分析

### 问题1：销售业绩排行统计不准确
**位置**：Admin.vue 的"销售业绩"标签页
**现象**：
- 销售用户的订单数、客户数、总销售额均显示为0
- 平均订单额显示为 NaN
- 原因：`admin-enhanced.js` 中 `/sales-performance` 接口的 SQL 查询条件错误，使用了 `u.id = o.user_id` 而非 `u.id = o.sales_user_id`

### 问题2：数据分析页面数据不完整
**位置**：Analytics.vue 页面
**现象**：
- 热门商品 TOP 10 和活跃用户 TOP 10 区域没有数据
- 前端调用 `api.analysis.getAnalytics()` 但此接口在 API 配置中不存在
- 实际正确的接口应该是 `api.analysis.getOverview()`

## 目标
1. 修复销售业绩排行的 SQL 查询，确保每个销售用户负责的订单被正确统计
2. 修复数据分析页面的 API 调用，确保热门商品和活跃用户数据正常显示
3. 保持现有代码的一致性和可维护性

## 非目标（超出范围）
- 不重新设计整个数据分析功能
- 不修改其他未涉及的页面或功能

## 背景与上下文
- 项目已有完整的数据库和路由结构
- `analysis-enhanced.js` 已有正确的数据查询接口 `/`，返回包含 overview、sales_trend、category_distribution、top_products、top_customers 的完整数据
- 前端 API 配置在 `frontend/src/api/index.js` 中
- 数据库中订单表使用 `sales_user_id` 字段标识负责该订单的销售人员

## 功能需求
### FR-1：修复销售业绩排行统计
- 修改 `admin-enhanced.js` 中的 `/sales-performance` 接口
- 修正 SQL 查询中的连接条件，从 `o.user_id` 改为 `o.sales_user_id`
- 增加 `avg_order_value` 字段返回给前端，避免前端 NaN 问题
- 确保接口权限只允许 admin 访问

### FR-2：修复数据分析页面数据展示
- 修改 `Analytics.vue` 中的 API 调用，使用正确的接口
- 确保页面能正确获取并展示 `top_products` 和 `top_customers` 数据
- 确保销售趋势图表和商品分类占比图表能正确渲染

## 非功能需求
### NFR-1：兼容性
- 修改必须与现有代码兼容，不破坏现有功能

### NFR-2：性能
- 查询性能不受影响，保持原有的响应速度

### NFR-3：可维护性
- 代码风格与现有代码保持一致
- 不引入不必要的复杂度

## 约束条件
- **技术栈**：Node.js、Express、Vue 3、MySQL
- **依赖**：需要数据库中已有订单和产品数据用于测试
- **权限**：只有 admin 用户可以访问相关管理功能

## 假设
1. 数据库结构未发生变化，`orders` 表有 `sales_user_id` 字段
2. 现有 API 认证和权限机制正常工作
3. 前端路由和组件结构已正确设置

## 验收标准

### AC-1：销售业绩排行正确显示数据
- **前置条件**：
  - 使用 admin 账号登录系统
  - 数据库中存在至少一个 sales 用户
  - 该 sales 用户有已支付/已发货/已完成的订单
- **操作步骤**：
  1. 进入"系统管理" → "销售业绩"标签页
  2. 查看销售业绩排行表格
- **预期结果**：
  - 销售员的订单数正确显示
  - 客户数（购买过该销售负责商品的不同用户数）正确显示
  - 总销售额正确显示（¥xxxx.xx 格式）
  - 平均订单额正确显示（不为 NaN）
- **验证方式**：人工测试 + 数据库查询比对
- **备注**：确保销售额和订单数与数据库中的实际数据一致

### AC-2：数据分析页面完整显示所有数据
- **前置条件**：
  - 使用 admin 账号登录系统
  - 数据库中存在订单和商品数据
- **操作步骤**：
  1. 点击导航栏中的"数据分析"
  2. 查看页面上的所有数据区域
- **预期结果**：
  - 顶部统计卡片显示正确数据（总销售额、订单总数、活跃用户、客单价）
  - 销售趋势图表正确渲染（12个月份数据）
  - 商品分类占比图表正确显示各分类的销售占比
  - 热门商品 TOP 10 正确显示前10个销量最高的商品
  - 活跃用户 TOP 10 正确显示前10个消费最高的用户
- **验证方式**：人工测试
- **备注**：确认数据与数据库实际情况一致

### AC-3：API 响应格式正确
- **前置条件**：
  - 服务器正常运行
  - 使用有效的 admin token
- **操作步骤**：
  1. 调用 `/admin/sales-performance` 接口
  2. 调用 `/analysis` 接口
- **预期结果**：
  - 两个接口都返回 HTTP 200 状态码
  - 返回的 JSON 数据结构正确
  - 无错误信息
- **验证方式**：程序测试
- **备注**：可以使用 Postman 或编写自动化测试
