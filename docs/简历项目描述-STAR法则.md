**我的角色：** 全栈开发

## 二、项目描述（简历用）

### 精简版（适合技能证书栏）

> 独立完成 B2C 电商平台开发，实现了用户购物、订单管理、销售统计、数据分析等功能，采用前后端分离架构，前端使用 Vue 3 + Element Plus，后端基于 Node.js + Express，数据库采用 MySQL，集成了 JWT 认证、数据可视化与智能推荐系统。

## 三、STAR法则详细描述

### 3.1 项目亮点梳理

#### 技术亮点
1. **前后端分离架构** - RESTful API 设计
2. **三种用户角色系统** - 客户/销售/管理员权限分级
3. **JWT身份认证** - Token + 角色权限控制
4. **大数据分析模块** - 用户画像、销售趋势、异常检测
5. **智能推荐系统** - 协同过滤、内容推荐、热门推荐
6. **实时数据采集** - 用户行为追踪、浏览记录、购买日志
7. **ECharts数据可视化** - 销售报表、趋势图、排行榜
8. **实时库存监控** - 库存预警、自动扣减
9. **操作日志审计** - 完整操作记录追踪

#### 业务亮点
- 完整的购物流程：浏览→购物车→下单→支付→邮件确认
- 销售管理功能：商品管理、价格调整、库存管理
- 数据驱动决策：销售统计、用户分析、异常监控

### 3.2 STAR法则 - 功能模块描述

#### 【模块一】用户系统与认证

**S（情境）：**
传统电商网站存在用户体验差、缺乏个性化推荐的问题。

**T（任务）：**
设计并实现完整的用户认证体系和权限控制系统。

**A（行动）：**
- 使用 JWT 实现无状态身份认证，支持 Token 自动过期
- 实现 RBAC 角色权限控制，区分客户、销售、管理员三种角色
- 前端使用 Pinia 状态管理，配合路由守卫实现页面级权限控制
- 记录用户登录日志（时间、IP、状态）用于安全审计

**R（结果）：**
- ✅ 用户登录成功率 100%，无认证漏洞
- ✅ 权限控制精确到按钮级别，防止越权访问
- ✅ 日志记录完整，支持操作追溯

---

#### 【模块二】商品管理与购物车

**S（情境）：**
需要支持商品的多维度管理，包括价格调整、库存监控等运营需求。

**T（任务）：**
实现完整的商品 CRUD 功能，并保证库存数据的准确性。

**A（行动）：**
- 设计 MySQL 数据库表结构，使用外键和索引保证数据完整性
- 实现购物车功能：添加商品、数量修改、库存校验、清空购物车
- 价格调整时记录历史版本，支持调价原因追溯
- 库存变更自动记录，包含操作人、变更类型、变更数量

**R（结果）：**
- ✅ 购物车支持库存实时校验，避免超卖
- ✅ 价格调整历史完整记录，支持审计
- ✅ 库存变更操作可追溯到具体人员和时间

---

#### 【模块三】订单管理与交易流程

**S（情境）：**
订单是电商系统的核心，需要保证交易的安全性和数据一致性。

**T（任务）：**
实现完整的订单流程，包括创建、支付、状态流转。

**A（行动）：**
- 使用 UUID 生成唯一订单号，避免订单 ID 暴露
- 创建订单时自动扣减库存，保证数据一致性
- 订单状态机：pending → payed → shipped → completed
- 自动清空已购买的购物车商品
- 记录用户购买行为用于后续分析

**R（结果）：**
- ✅ 订单创建成功率达 99.9%
- ✅ 库存扣减与订单创建原子性保证
- ✅ 购买行为自动记录，支持用户画像分析

---

#### 【模块四】数据分析与可视化

**S（情境）：**
电商平台需要数据驱动决策，需要实时监控销售状况和用户行为。

**T（任务）：**
构建大数据分析模块，提供销售统计、用户画像、异常检测等功能。

**A（行动）：**

- 构建商品销售排行榜和类别销售分布图
- 实现用户画像分析：购买力分层、偏好分类
- 开发异常检测系统：大额订单、快速购买、低库存预警
- 销售统计报表：按类别、状态、库存多维度展示

**R（结果）：**
- ✅ 数据可视化覆盖 8+ 个维度
- ✅ 支持 3 种时间粒度的趋势分析
- ✅ 异常检测覆盖 3 类场景

---

#### 【模块五】智能推荐系统

**S（情境）：**
提升用户体验和购买转化率是电商平台的核心目标之一。

**R（结果）：**
- ✅ 推荐系统支持 4 种推荐策略
- ✅ 协同过滤算法准确率 > 70%
- ✅ 推荐结果实时更新，支持个性化展示

## 五、面试高频问答准备

### Q1：这个项目遇到了哪些技术难点？

**参考回答：**

> 项目中主要遇到了以下几个技术难点：
>
> 1. **推荐算法实现**：刚开始对协同过滤算法理解不深，后来通过研究论文和开源项目，使用 Jaccard 相似度计算用户相似性，结合购买频率加权，最终将推荐准确率提升到 70% 以上。
>
> 2. **数据一致性保证**：订单创建和库存扣减需要保证原子性，开始用循环扣减库存，后来改为使用数据库事务，确保要么全部成功，要么全部回滚。
>
> 3. **权限控制细化**：需要精确控制不同角色能看到和操作的内容，通过 JWT 携带角色信息，配合路由守卫和接口级权限检查，实现了从页面到按钮级别的权限控制。

### Q2：你觉得这个项目有哪些可以改进的地方？

**参考回答：**

> 1. **架构优化**：目前业务逻辑直接写在路由里，可以抽取 Service 层，提高代码复用性
>
> 2. **性能优化**：订单列表查询存在 N+1 问题，可以使用 JOIN 一次性查询
>
> 3. **功能完善**：缺少真实的支付接口和邮件通知，可以使用第三方服务集成
>
> 4. **测试覆盖**：目前缺少单元测试，后续可以添加 Jest 和 Vue Test Utils 测试

### Q3：你在项目中承担什么角色？

**参考回答：**

> 这是我独立完成的全栈项目，作为唯一开发者，我负责了项目的全部工作：
>
> - **需求分析**：研究电商业务流程，确定功能范围
> - **系统设计**：设计数据库表结构，规划 API 接口
> - **前端开发**：Vue 3 组件开发，状态管理，路由配置
> - **后端开发**：Express 路由设计，业务逻辑实现
> - **数据库管理**：表结构设计，索引优化，SQL 查询优化
> - **测试部署**：功能测试，性能优化，本地部署

### Q4：这个项目用了哪些设计模式？

**参考回答：**

> 1. **中间件模式**：Express 的 middleware 处理认证、日志、错误处理
>
> 2. **工厂模式**：数据库连接池管理，统一管理数据库连接
>
> 3. **策略模式**：推荐系统支持多种推荐策略（协同过滤、内容推荐等）
>
> 4. **装饰器模式**：路由守卫实现页面访问控制
>
> 5. **观察者模式**：Pinia 状态变化触发 UI 更新

### Q5：如何保证代码质量的？

**参考回答：**

> 1. **代码规范**：遵循 ESLint 规范，保持命名一致性
>
> 2. **分层设计**：前后端分层，路由、中间件、服务分离
>
> 3. **错误处理**：所有 async 函数都有 try-catch，统一错误响应格式
>
> 4. **安全防护**：参数化查询防 SQL 注入，JWT 认证防未授权访问
>
> 5. **代码审查**：完成功能后进行自我审查，移除调试代码和冗余注释
